### 🛑 問題診斷 (Diagnosis)

主要有兩個核心問題導致 API 請求失敗：

1. **CORS 封鎖 (CORS Policy Blocked)**
* **錯誤訊息：** `Access to fetch at '...' from origin 'https://penter405.github.io' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present...`
* **原因：** 您的前端網站架設在 **GitHub Pages** (`penter405.github.io`)，但後端 API 架設在 **Vercel**。Vercel 的伺服器預設不允許來自 GitHub Pages 的請求。您必須在 Vercel 後端設定中明確允許 `https://penter405.github.io` 這個網域。


2. **401 Unauthorized (未授權)**
* **Log 證據：** `requestMethod":"OPTIONS", "responseStatusCode":401`
* **原因：** 瀏覽器在發送正式請求前會發送一個 `OPTIONS` 預檢請求（Preflight request）。Vercel 直接回傳了 401 錯誤。這通常是因為：
* 後端沒有正確處理 `OPTIONS` 請求。
* Vercel 專案開啟了某些保護機制（如 Vercel Authentication），或者 API Key 驗證邏輯在 CORS 檢查之前就先擋下了請求。

### 💡 技術補充：如果您能自己修改程式碼

如果您有權限修改 Vercel 的程式碼（例如使用 Next.js），請在 `next.config.js` 加入以下設定來解決 CORS 問題：

```javascript
// next.config.js
module.exports = {
  async headers() {
    return [
      {
        // 配對所有 API 路徑
        source: "/api/:path*",
        headers: [
          { key: "Access-Control-Allow-Credentials", value: "true" },
          { key: "Access-Control-Allow-Origin", value: "https://penter405.github.io" }, // 或用 "*" (不推薦用於生產環境)
          { key: "Access-Control-Allow-Methods", value: "GET,OPTIONS,PATCH,DELETE,POST,PUT" },
          { key: "Access-Control-Allow-Headers", value: "X-CSRF-Token, X-Requested-With, Accept, Accept-Version, Content-Length, Content-MD5, Content-Type, Date, X-Api-Version" },
        ]
      }
    ]
  }
};

```
